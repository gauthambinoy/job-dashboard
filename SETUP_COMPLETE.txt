SCHEMA DETAILS
================================================================================

4 Tables to be created:

user_profiles
  - Stores user profile data
  - Fields: id, user_id, skills[], experience_years, education, salary_min/max,
            target_countries[], availability, profile_updated_date, created_at
  - Primary key: id
  - Unique constraint: user_id

jobs
  - Stores job postings with full job description and extracted data
  - Fields: id, company, title, location, country, salary_min/max, currency,
            jd_full_text, original_url, source, extracted_skills_required[],
            extracted_skills_nice_to_have[], experience_level, degree_required,
            soft_skills[], job_type, posted_date, cluster_id, match_score,
            created_at, updated_at
  - Primary key: id
  - Indexes: title, country+match_score, country, source, cluster_id
  - Unique constraint: original_url

job_clusters
  - Groups similar jobs together by domain
  - Fields: id, domain, job_ids[], avg_match_score, skill_vector (JSONB),
            cv_suggestion, required_skills_consolidated[], created_at
  - Primary key: id (VARCHAR)

saved_jobs
  - Tracks job applications and saved jobs for users
  - Fields: id, user_id, job_id, cluster_id, status, cv_variant_used, notes,
            date_saved, date_applied, interview_date, result_notes,
            created_at, updated_at
  - Primary key: id
  - Foreign keys: user_id → user_profiles.user_id, job_id → jobs.id
  - Indexes: user_id+status, user_id, cluster_id
