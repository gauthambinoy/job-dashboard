RAILWAY DEPLOYMENT CHECKLIST (Recommended for fastest setup)
================================================================================

PRE-DEPLOYMENT (Estimated time: 5 minutes)
[ ] GitHub account created and verified
[ ] Code pushed to GitHub repository
[ ] Railway account created (sign up at railway.app)
[ ] GitHub authorization granted to Railway

DEPLOYMENT SETUP (Estimated time: 25 minutes)
[ ] Create new Railway project from GitHub
[ ] PostgreSQL service created
  [ ] DATABASE_URL copied and saved
[ ] Backend service created and configured
  [ ] Root directory set to: backend
  [ ] Build command: npm run build
  [ ] Start command: node dist/index.js
  [ ] Environment variables set:
    [ ] DATABASE_URL (from PostgreSQL)
    [ ] API_PORT=5000
    [ ] NODE_ENV=production
    [ ] JWT_SECRET (generated)
    [ ] FRONTEND_URL=http://localhost:3000 (temporary)
  [ ] Service deployed successfully (green status)
  [ ] Backend URL copied and saved
[ ] Frontend service created and configured
  [ ] Root directory set to: frontend
  [ ] Build command: npm run build
  [ ] Start command: npm start
  [ ] Environment variables set:
    [ ] NEXT_PUBLIC_API_URL=https://[backend-url]/api
  [ ] Service deployed successfully (green status)
  [ ] Frontend URL copied and saved
[ ] Backend FRONTEND_URL variable updated
  [ ] FRONTEND_URL=https://[frontend-url]
  [ ] Service redeployed (triggered automatically)

POST-DEPLOYMENT VERIFICATION
[ ] Backend health check passes
  Command: curl https://[backend-url]/health
  Expected: {"status":"healthy","database":"connected"...}
[ ] Frontend loads without errors
  [ ] Open https://[frontend-url]
  [ ] No console errors (F12 → Console)
  [ ] No CORS errors
[ ] API connectivity working
  [ ] API calls from frontend reach backend successfully
  [ ] Data is persisting correctly
[ ] Database connectivity confirmed
  [ ] Data saved to database successfully
  [ ] Data retrieval working

OPTIONAL POST-DEPLOYMENT
[ ] Custom domain added (if applicable)
[ ] Monitoring/alerts configured in Railway
[ ] Auto-deploy on git push verified

COMMON ISSUES & SOLUTIONS
================================================================================

Issue: Backend won't start
[ ] Check DATABASE_URL is correct
[ ] Verify database is accessible
[ ] Check NODE_ENV is set correctly
[ ] Review build logs for errors
Solution: Check logs and error messages, fix issues, redeploy

Issue: Frontend can't reach API
[ ] Verify NEXT_PUBLIC_API_URL is correct
[ ] Check CORS configuration in backend
[ ] Ensure backend is running
[ ] Check network connectivity
Solution: Update environment variables, verify backend health, check logs

Issue: Database connection fails
[ ] Verify DATABASE_URL format
[ ] Check database credentials
[ ] Ensure database is running
[ ] Verify security group rules
Solution: Update DATABASE_URL, restart database, check connectivity

Issue: Static assets return 404
[ ] Verify build output structure
[ ] Check public folder included
[ ] Verify CDN/S3 configuration
[ ] Check file permissions
Solution: Rebuild and redeploy, verify asset paths

Issue: High latency or timeouts
[ ] Check resource allocation
[ ] Review database query performance
[ ] Check network connectivity
[ ] Monitor service health
Solution: Scale up resources, optimize queries, investigate logs
