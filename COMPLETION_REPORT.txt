================================================================================
CLUSTERING ALGORITHM
================================================================================

HOW IT WORKS:
  1. Extract skill vector for each job
  2. Calculate cosine similarity between jobs
  3. Group jobs with 85%+ similarity
  4. Identify domain/category for each cluster
  5. Consolidate required skills per cluster
  6. Generate CV suggestions

EXPECTED RESULTS:
  - 15-20 clusters from 110+ jobs
  - 80%+ clustering rate
  - Domain identification (Backend/Frontend/Data/DevOps/etc.)
  - CV suggestions for cluster groups

EXAMPLE CLUSTER:
  Cluster ID: C-001
  Domain: Backend Engineering
  Jobs: 8
  Skills: Python, PostgreSQL, Docker, AWS, FastAPI
  Suggestion: "Use 1 CV for all 8 jobs"

================================================================================
MATCHING ENGINE
================================================================================

SCORING FACTORS (0-100 each):
  1. Skills Match: How many required skills user has
  2. Experience Match: Years vs job requirement
  3. Salary Match: User expectations vs job salary
  4. Location Match: Job location vs user preferences
  5. Education Match: User education vs job requirement

TOTAL SCORE: Weighted average of 5 factors (0-100)

USAGE:
  POST /api/matching/calculate
  Input: User profile (skills, experience, salary, countries)
  Output: Match scores for all jobs + recommendations
